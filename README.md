# Aurelia — ArtAI

Prototype for an art-commissioning site, originally exported from Claude Design
as a single bundled `index.html` and since unpacked into an editable project.

## Layout

```
index.html                 the page — edit this
dc-runtime.js              Claude Design's rendering runtime (generated; don't edit)
vendor/                    React + ReactDOM UMD builds, inlined into the bundle
assets/
  portraits/               6 artist portraits
  works/                   54 artworks, <artist>-NN.jpg
  fonts/                   Cormorant Garamond woff2 subsets
  texture/grain.png        paper-grain overlay
tools/
  build.js                 re-pack everything into one shareable file
  fetch-art.js             re-download the artwork set from Wikimedia
  shell.html               the original bundler's loader (used by build.js)
  art-manifest.json        which source image landed in which slot
dist/index.html            build output — single self-contained file (~41 MB)
CREDITS.md                 per-image sources
```

## Viewing it

`index.html` references assets by relative path, so it needs a server — opening
the file directly will not load the images:

```bash
python -m http.server 8765
# then open http://127.0.0.1:8765/index.html
```

The unbundled page pulls React from unpkg, so it needs a network connection.
`dist/index.html` does not — everything is inlined there.

## How the page is structured

The markup is Claude Design's `<x-dc>` template DSL: `{{ expr }}` interpolation,
`<sc-for list="{{ … }}" as="x">` loops, `<sc-if value="{{ … }}">` conditionals,
`sc-camel-on-click` handlers, and `style-hover` for hover states. All the data
lives in the `class Component extends DCLogic` block near the bottom of
`index.html`.

### Adding or changing an artist

Find the `get artists()` getter. Each entry looks like:

```js
{id:0, name:'Vincent van Gogh', style:'Post-Impressionismo', place:'Arles, Francia',
 era:1888, medium:'Olio su tela', tagline:'…', bio:'…', philosophy:'…',
 tags:['Impasto','Colore espressivo','Paesaggio','Ritrattistica'],
 portrait:'assets/portraits/van-gogh.jpg',
 works:[
   {t:'Notte stellata', y:1889, img:'assets/works/van-gogh-01.jpg', r:'1280 / 1014'},
   … 9 works total
 ]}
```

- `portrait` — path to the artist image (displayed at 4:5, `object-fit:cover`).
- `t` / `y` / `img` — title, year, image path.
- `r` — the image's **real** pixel ratio, e.g. `'1280 / 1014'`. The home-page
  masonry uses it as the CSS `aspect-ratio` so paintings are never cropped. If
  you swap an image, update `r` to match or that tile will letterbox.

Each artist needs exactly 9 works — `buildArtworks()` indexes fixed arrays of
dimensions and prices with `i % 9`.

### Where images appear

| Placement | Fit |
|---|---|
| Featured-artist cards (home) | 4:5, `cover` |
| Artist profile portrait | 4:5, `cover` |
| Home gallery masonry | real ratio from `r`, uncropped |
| Artist profile works grid | 4:3 frame, `contain` — full painting, matted |

## Building the single-file version

```bash
node tools/build.js      # -> dist/index.html
```

Every locally-referenced asset is base64'd into the bundler manifest and each
path in the markup is swapped for the asset's uuid. The result opens straight
from disk with no server and no network. It is ~41 MB — base64 costs ~33% on top
of 31 MB of images — so it's for sharing, not for hosting.

Two things `build.js` must keep doing, both of which broke the bundle in
testing:

- Scan for asset paths in **single-quoted JS strings**, not just HTML
  attributes. The artwork paths live in the component data (`img:'assets/…'`).
- Escape `</` as `</` in the embedded JSON. The template contains the
  page's own `</script>`, which otherwise closes the data block early.

## Re-downloading the images

```bash
node tools/fetch-art.js
```

Resumable — it skips anything already in `tools/art-manifest.json`, so a
rate-limit stop can just be re-run. Sources are listed in [CREDITS.md](CREDITS.md).

## Notes

- All artwork is public domain (see CREDITS.md), but the **prices and
  dimensions in the catalogue are invented prototype data** — they are not the
  real dimensions or any real valuation of these paintings.
- `dc-runtime.js` is generated by Claude Design. Don't hand-edit it; it will be
  overwritten if the design is ever re-exported.
